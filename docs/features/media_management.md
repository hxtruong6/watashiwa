# Feature Specification: Media Management (Image Upload)

## 1. Goal

Enable users to upload and manage images for their profile avatars and vocabulary/kanji cards. This enhances the learning experience by adding visual associations to study materials and personalizing the user profile.

## 2. Technical Architecture

### 2.1. Storage Provider

**Google Cloud Storage (GCS)**

- **Bucket Strategy**: Single public-read bucket (e.g., `watashi-jp-assets`) or proxied via Next.js for privacy.
- **Access Control**:
  - **Uploads**: Use **Signed URLs** generated by the server. This prevents exposing service account credentials to the client.
  - **Reads**: Public URLs (e.g., `https://storage.googleapis.com/...`) for performance.

### 2.2. Database Changes

- **User Model**: Add `avatarUrl` (String?).

* **VocabCard Model**: Add `imageUrl` (String?).
* **Kanji Model**: Add `imageUrl` (String?).
* _(Future)_: separate `Media` table if we need to track usage/references, but direct URL string is sufficient for MVP.

## 3. User Flows

### 3.1. Profile Avatar

1. **User** goes to Profile Settings.
2. Clicks on current avatar placeholder.
3. Selects image file.
4. **Client** requests Upload URL from Server.
5. **Server** generates GCS Signed URL (PUT).
6. **Client** uploads file directly to GCS.
7. **Client** calls Server to update `User.avatarUrl` with the new public URL.

### 3.2. Card Customization (Creator Mode)

1. **User** opens Deck Editor -> Add/Edit Card.
2. "Image" field available.
3. User uploads image (Drag & Drop).
4. System uploads to GCS (same signed URL flow).
5. Image preview shown on card.
6. Saving the card saves the `imageUrl`.

## 4. UI/UX Requirements

### 4.1. Components

- **`ImageUploader`**: Reusable component.
  - **Props**: `onUploadComplete(url: string)`, `currentImage`, `shape` (circle/rect).
  - **UI**:
    - Empty state: Icon + "Click to upload".
    - Loading state: Progress bar / Spinner.
    - Error state: Message (Size limit, type).

### 4.2. Pages Impacted

- **`/profile` (or `src/components/NavBar` for quick access)**: Add Avatar settings.

* **`/dashboard/decks/[id]/edit`**: Add Image upload to `VocabEditor` / `KanjiEditor`.
* **Flashcard View**: Render image if exists (Standard `FlashCard.tsx`).

## 5. Security & Validation

- **File Types**: `image/jpeg`, `image/png`, `image/webp`.

* **Max Size**: 5MB.
* **Path Naming**:
  - Avatars: `avatars/{userId}_{timestamp}.ext`
  - Cards: `cards/{deckId}/{timestamp}_{random}.ext`
